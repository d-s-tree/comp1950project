Our Proposed Site Improvements

With the combined efforts of our team, we will implement several new improvements for the current COMP 1950 website in order to bring it forward to modern day web standards.

With the implementation of HTML5 as the new W3C recommendation in 2014, re-working the COMP 1950 website utilizing HTML5 standards was a must do. We will utilize a BootStrap framework which will give us a solid foundation of both HTML5, CSS3 and jQuery functionality. Our layout and design use the new HTML5 tags throughout giving the webpage content greater semantic meaning than it had previously. This increased semantic meaning will allow for search engines to better interpret the content of the page which in turn will result in improved rankings on search engine algorithms.

Receiving an error 404 is never a positive experience for a user, but with a bit of work the experience can be improved. Our webpage will have a custom error 404 page implemented. This page will be branded appropriately to match the style and feel of the other website pages and will include several links that will properly lead the user to functioning pages and hopefully toward the content they were originally trying to reach. Because users will be more likely to click through links on our custom 404 page instead of hitting the back button immediately, our Search Engine Optimization (SEO) will likely be improved as well.

Occasionally users will desire a hard copy of the website content. A poorly print-styled page can lead to a negative user experience. Our pages will utilize unique styles specific to print jobs, ensuring that any unnecessary information (navigations, images not related to the main content, unnecessary colours, etc.) are excluded from the printout and content (paragraphs, images) are not cutting across page breaks.

Typically, websites will have certain styles that remain constant across all pages on a site. These are typically headers, navigations, and footers. If changes are needed to any of these sections, usually they need to be made on all pages. While not too concerning for websites with one or two pages, this can become problematic and time-consuming when working with websites with a larger number of pages. To resolve this, our page will use PHP scripting to insert the common sections of our webpage. With this implementation, it will ensure that if any changes are needed to be made to any of these sections in the future, they will only need to be made once to the master version being inserted to all through PHP.

A primary pain point about the current COMP 1950 website was the inability to quickly jump to the necessary section on a given lecture page. While bookmarks are present at the top of the page for fast navigation, if you are mid-page it requires scrolling back up to see the bookmarks or scroll through the page manually to find the information needed. To improve upon this experience, we have decided on one implementation through jQuery. This improvement is to implement ‘accordion-style’ sections. Each heading on a lecture page will denote a section. When first visiting a lecture page, all these sections will be collapsed, and the content that is usually seen under them will not be initially visible. As the content is collapsed, the user will be able to see most of (if not all) of the section headings. When clicked, these sections will expand, and the content will be visible. When clicked again, the content will collapse. This feature will ensure that all headings will be typically visible to the user and thus easier to find its current form. A secondary implementation that we believe will make the navigation of content easier for the user is a fixed header navigation at the top of the page. With this, users will be able to quickly access the links to any of the other lectures or pages on the site without the need of scrolling back to the top of the page.

Over the past few decades search engine’s have become more and more refined in assessing webpages and ranking them accordingly. We will attempt to optimize our new webpage with the best practices regarding search engine rankings. This will include:

  1. Having well-structured page titles, accurately describing the page in a brief statement.
  2. Utilizing the ‘description’ meta tag, and ensuring that it is a accurate summary of the page content.
  3. Including a site map.
  4. As mentioned earlier, including a custom-made error 404 page.
  5. A respectable page load time.
  6. Well-structured page URLs that make the page hierarchy easy to understand and follow.

Online quizzes will also be implemented into the website, increasing its functionality. The links to these quizzes will be found directly on the lecture pages and will be formatted as a form in a visually pleasing way. Validators will be added throughout the form to ensure certain questions (such as multiple choice) cannot be left unfilled, and users will be redirected to a ‘form completed’ page on submission.
